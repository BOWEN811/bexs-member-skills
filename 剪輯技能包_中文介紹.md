## 兩格判斷法

### 第 1 格：短影片專用版本

用 `ig-short-video-marketing-editor`。

只要目標是 IG Reels、Shorts、TikTok、直式短影片、9:16 行銷短片，就用這個。

它裡面有兩個風格分支：

- **Podcast 訪談風格**：純訪談、Podcast 感、黑底高級畫面、主持人和受訪者要分開；正式字幕仍一律白字黑描邊。
- **直式拍螢幕預告風**：手機拍螢幕、手指指畫面、課程預告、會員公告、展示 Codex / PalmierPro / GitHub。

你可以這樣講：

```text
這支用短影片專用版本剪，打開專案給我看
```

```text
這支照 Podcast 訪談風格做，Rick 跟受訪者不要混，字幕不能漏，字幕白字黑描邊
```

```text
這支是直式拍螢幕預告，做成 IG Reels，先完整抽字幕理解全片，字幕白字黑描邊
```

### 第 2 格：YouTube 長片專案精修版

用 `youtube-longform-project-finisher`。

只要目標是 YouTube 長片、課程長片、16:9 訪談、長教學、長片專案整理，就用這個。

它處理的是長片專案，不是 IG 短片。

你可以這樣講：

```text
這支 16:9 長片用 YouTube 長片專案精修版，先完整抽字幕理解全片
```

```text
這支 YouTube 長片幫我整理字幕、音軌、章節，打開專案給我看
```

```text
這支課程長片幫我做專案精修，不要剪成短影音
```

## 強制規則：不能只跑 CLI 版本

這是 Rick 特別要求的剪輯標準。

不管用哪個剪輯 skill，都不能只在背景跑完，丟一個 mp4 檔案就結束。

每次都必須：

- 先抽取或取得整支影片的完整字幕/逐字稿，不只抽片段。
- 先讀完整字幕，理解整部影片的主旨、段落、重點、可刪段落與不可刪段落，再決定怎麼剪。
- 字幕理解一律使用 `.srt` / `.vtt` / `.ass` / `.txt` / `transcript.json` / Whisper 逐字稿；不得把整支影片切成大量 PNG 後靠 OCR 讀字幕。
- PNG / 截圖只用於 PalmierPro 視覺抽查：畫面比例、安全區、字幕位置、白字黑描邊、是否擋臉或遮住重點；不做全片逐秒截圖。
- 省 PNG 不等於跳過字幕樣式驗收；正式字幕仍必須白字黑描邊，不能改成彩色字幕或大黑底框。
- 不可以只看檔名、前 30 秒、聲波或畫面猜內容後直接剪。
- 產生 `.palmier` 專案並打開 PalmierPro。
- 讓 Rick 在 PalmierPro timeline 上看得到 V 軌、A 軌、字幕素材/字幕軌與分段。
- A 軌要看得到 waveform。
- 字幕不能只燒進 mp4，必須有字幕素材、字幕段或 overlay 檔在軌道上。
- 正式字幕一律白色文字 + 黑色描邊/外框；黑色描邊不是大黑底框。
- preview/review 頁只能當補充，不能取代 PalmierPro。
- 音軌不能中途沒聲音。
- 字幕不能漏，也不能跟聲音錯位。
- 如果是訪談，主持人和受訪者不能混在一起。
- 如果是短影片，畫面與字幕要放在 IG 安全區內。
- 完成後要在 CLI / Claude App / GPT App 回覆「已完成：剪輯完成。」並附 3-5 條可直接複製的建議生成提示詞。
- 完成後要全自動接 `rick-yt-publish` 上架包：CLI / Claude Code / Codex 可直接呼叫本地 skill；Claude App / GPT App 就在同一則最後回覆直接輸出同等上架包。
- 上架包至少要包含標題、簡介、章節或段落、hashtag、置頂留言建議、封面提示詞。
- 不要自動真的上傳或發布到 YouTube / IG / TikTok，除非 Rick 另外明講。

簡單講：**要做出可以檢查的 PalmierPro 剪輯專案，不是只交一個黑箱成片。**

## 短影片專用版本細節

Skill：`ig-short-video-marketing-editor`

### 適合

- IG Reels
- YouTube Shorts
- TikTok
- 9:16 直式短影片
- 純訪談黑底字幕短片
- 手機直拍螢幕預告
- 課程公告
- 會員公告
- 自動剪輯短影片

### 兩個內部分支

| 分支 | 什麼時候用 | 重點 |
|---|---|---|
| Podcast 訪談風格 | 純訪談、Podcast、沒有露臉 | 黑底高級畫面、用姓名標籤/排版分 speaker、逐字稿不能漏、字幕白字黑描邊 |
| 直式拍螢幕預告風 | 手機拍螢幕、課程預告、展示工具 | 保留真實感、字幕白字黑描邊、上方標題放安全區 |

### 硬規則

- 9:16，1080x1920。
- 字幕繁體中文，正式字幕一律白字黑描邊。
- 剪輯前先完整抽字幕並理解全片。
- PNG 只做少量視覺抽查，短片通常不超過 8-12 張；不得用全量 PNG/OCR 取代逐字稿。
- 必須打開 PalmierPro 專案。
- 必須能看到音軌 waveform、字幕素材/字幕軌與分段。
- 字幕不能只燒進 mp4，必須在 timeline 上可檢查。
- 不可只跑 CLI / headless 版本。
- 完成後必須回覆建議生成提示詞，並自動產出短影音上架包。
- IG 安全區保守值：左 `92px`、右 `988px`、上 `170px`、下 `1640px`。

## YouTube 長片專案精修版細節

Skill：`youtube-longform-project-finisher`

### 適合

- YouTube 長片
- 16:9 課程影片
- 長訪談
- 長教學
- 長片字幕整理
- 長片音軌修整
- YouTube chapter / 章節
- 片頭、片尾、下三分之一、B-roll、長片包裝

### 不適合

- IG Reels
- Shorts
- TikTok
- Podcast 黑底訪談短片
- 手機拍螢幕預告

這些全部回到 `ig-short-video-marketing-editor`。

### 硬規則

- 16:9，不要錯裁成 9:16。
- 保留長片敘事，不要剪成短影音節奏。
- 剪輯前先完整抽字幕並理解全片。
- PNG 只做少量視覺抽查；長片以片頭、章節/大段落、字幕密集處、片尾為主，不做全片逐秒截圖。
- 正式字幕一律白字黑描邊。
- 必須打開 PalmierPro 專案。
- 必須能看到音軌 waveform、字幕素材/字幕軌、章節與片段切點。
- 字幕不能只燒進 mp4，必須在 timeline 上可檢查。
- 不可只跑 CLI / headless 版本。
- 完成後必須回覆建議生成提示詞，並自動產出 YouTube 長片上架包。

## 影片上架包不是剪輯入口

`rick-yt-publish` 只用在影片剪完後，不能拿來當剪輯入口。

現在兩個剪輯 skill 已經全自動接上它：剪輯、匯出、打開 PalmierPro、完成檢查後，會直接產出上架包，不需要 Rick 再補一句「幫我做上架包」。

它負責：

- YouTube 標題
- 影片簡介
- 章節文字
- hashtag
- 封面標題
- 封面提示詞

如果你已經有一支剪好的影片，也可以單獨叫它：

你可以這樣講：

```text
這支影片剪完了，給我標題、內文、封面提示詞
```

它不是剪輯工具，不要拿來剪片，也不會自動真的把影片上傳或發布。

## 學生安裝方式

下載這個 repo 後，把想用的 skill 內層英文資料夾複製到 `~/.claude/skills/`。

只安裝短影片專用版本：

```bash
cp -r "skills/進階剪輯-IG短影片行銷自動剪輯-ig-short-video-marketing-editor/ig-short-video-marketing-editor" ~/.claude/skills/
```

只安裝 YouTube 長片專案精修版：

```bash
cp -r "skills/進階剪輯-YouTube長片專案精修-youtube-longform-project-finisher/youtube-longform-project-finisher" ~/.claude/skills/
```

全部安裝：

```bash
for d in skills/*/*/; do cp -r "$d" ~/.claude/skills/; done
```
