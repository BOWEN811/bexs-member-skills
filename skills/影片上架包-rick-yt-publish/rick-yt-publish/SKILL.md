# 🎬 Rick 影片上架助手 · rick-yt-publish

**這個幫你:把剛錄好、跑完 Whisper 的字幕修乾淨(繁體+錯字+工具名),再一口氣產出 YouTube 標題、簡介、封面提示詞 —— 一支影片從「毛片」到「可上架」一條龍。**

> 固化來源:2026-06-22 這次幫 Rick 處理 1 支黃金分析 + 2 堂 AI 教學影片,踩過簡體外漏、千問亂套術語、AI 工具名全錯三大雷。

---

## 中文 SKILL 題型關鍵詞

看到這些需求時，用這個 skill：

- **影片上架包**：YouTube 標題、簡介、章節、hashtag、封面提示詞。
- **字幕清稿上架**：字幕變繁體、字幕校正、修錯字、工具名修正、交易術語校正。
- **封面文案**：封面標題、封面副標、封面提示詞、我要丟 GPT 網頁版設計封面。
- **發片前整理**：這支要上 YouTube、幫我做上架資料、標題內文建議。

不要用在：

- **剪出短影音本體**：依情境用 `video-shorts`、`interview-reels-dialogue-editor` 或 `palmier-pro-shorts-finisher`。
- **只下載別人的 YouTube 做 Podcast**：改用 `yt-to-podcast`。

## 什麼時候用
- Rick 丟一個 `.srt`(或影片檔)說「這支做一下 YouTube / 做標題簡介 / 做封面」。
- Rick 說「字幕變簡體了 / 改錯字 / 工具名修一下」。
- 通常檔案在 `~/Desktop/毛片/`,一次可能有好幾支(交易 + AI 教學混著)。

## 第一步:先判斷「影片類型」(決定走哪條校正線)

| 類型 | 特徵 | 校正策略 |
|---|---|---|
| **交易類** | 黃金/盤勢/K棒/SOP/壓力位… | 可跑 `ai_polish.py`(千問+BEXS術語表,已含防幻覺鎖)+ 人工補表 C |
| **AI/科技教學** | Claude/Cursor/skill/模型/終端機… | **絕對不要跑 ai_polish**;改用對照表 B 確定性替換 + 人工補 |
| **其他/一般** | 日常、訪談 | 只做繁體化 + 明顯錯字,不套任何術語表 |

> 對照表在 `references/term-correction-tables.md`(A 共用簡繁 / B AI工具名 / C 交易術語)。

## 標準流程

### 階段 1 — 字幕校正
1. **先備份**:複製成 `xxx.before_<本次>.srt`(出事可還原)。
2. **全檔繁體化**(表 A):`opencc` `s2tw` 轉**整個檔**,不是只轉千問動過的句子。
   → 這步抓 Whisper 自己吐出來的簡體段落。
3. **依類型校錯字**:
   - `ai_polish.py`(2026-06-22 起)**會自己偵測類型分流**,跑 `python3 ai_polish.py xxx.srt` 即可:
     - 判**交易** → 千問 + BEXS 術語表 + 防幻覺鎖(本地千問要先開,model `Qwen3-Next-80B`,port 11434)→ 再人工補表 C。
     - 判 **AI/科技** → **自動跳過千問**,改用內建確定性工具名表(= 表 B)+ opencc;並把疑似被亂套的 `BEXS` 句**列出來提醒人工確認**(該改 Claude 還是 Ollama)。
   - 偵測邏輯:數交易訊號 vs AI 訊號,AI 明顯多且 ≥3 → AI;否則交易(主場)。誤判時可不靠這支,直接手動跑表 B。
   - **BEXS 三義字**(學院/示範漢堡品牌/被亂套)→ 用「整句唯一字串」逐條換,**永遠不可全域 replace**;羊駝(Ollama)、龍蝦(OpenClaw)暱稱保留。
4. **驗證**:掃殘留簡體(opencc 前後比對)、grep 關鍵錯字確認、抽查幾句語意對不對。
5. **回報**:列改了哪幾類、幾條;**沒把握的諧音字標「最佳猜測,請 Rick 聽一下確認」**,不要硬改裝懂。

### 階段 2 — YouTube 上架包(三件)
1. **標題** 3–4 個選項:鉤子放前段(恐懼/好奇/可學到什麼),壓 ~30 字內,推薦第一個。
2. **簡介**:開場鉤子 → `📌 重點` → `💡 一句話心法` → `⏱ 章節`(從 SRT 時間戳抓,第一個必 `00:00`)→ 免責聲明 → `#hashtag`。連結位置留給 Rick 自己補。
3. **封面提示詞**:中文版 + 英文版雙份;主標/副標/角落貼標分層;深色科技/金融質感。

## ⚠️ 踩過的雷(具體怎麼避)

1. **千問校正後字幕變簡體** → 根因:`ai_polish.py` 階段5 的千問常順手吐簡體。**光靠 prompt 叫它講繁體沒用**。解:跑完用 `opencc s2tw` **硬轉一道**,而且套用到**全部句子(含千問退回原文那批)**,因為簡體有時是 Whisper 來源就錯、沒經過千問。(此修已進 ai_polish.py)

2. **千問把非交易影片亂套 BEXS 術語** → 最慘的雷:AI 教學影片裡,千問把 Rick 講的 **Ollama 硬改成「BEXS」**、Claude 留成 Cloud。根因:BEXS 術語表是給交易設計的,套到別的主題就幻覺。解(三層,已全進 ai_polish.py):**①自動偵測類型,AI 影片自動跳過千問改走確定性表 ②防幻覺鎖 `GUARD_TERMS` 禁注入術語 + `SIMILARITY_FLOOR=0.55` 擋大改寫 ③BEXS 句列出提醒人工確認**。

3. **千問回傳行數對不上 → 整批退回原文 → 那批的簡體沒被轉** → 解:見雷1,opencc 全套兜底。

4. **AI 工具名 Whisper 全錯**(Cloud=Claude、GBT=GPT、中端機=終端機、OpenCloud=OpenClaw…) → 解:用**確定性對照表 B**,**不丟千問**(丟了只會更糟)。`Cloud`→`Claude` 但「雲端」中文不要動。

5. **BEXS 在 AI 影片是三義字**(學院/示範漢堡品牌/被亂套的 Ollama或Claude) → 解:**不能全域 replace**,用整句唯一字串逐條換;`羊駝`(Ollama)、`龍蝦`(OpenClaw)是 Rick 暱稱要保留。

6. **opencc 用 `s2tw` 不要用 `s2twp`** → s2twp 會連用詞一起改(可能動到專有名詞),s2tw 只轉字元最保險。

7. **GPT 生封面的中文字常缺筆畫/變形**(裝、調、複 這種) → 解:叫 GPT **只生背景、留白不放字**,中文用本地工具(PIL/Canva)精準疊上去;或提醒「文字務必清晰正確」重生。

8. **章節時間只是字幕概略點** → 一定提醒 Rick 發片前快轉對一下;YouTube 第一個章節必須是 `00:00` 才生效。

## 做到 Rick 滿意的協作標準
- **一步一步、邊做邊收**:Rick 會在中途補話,持續接收融進工作流,不要等他講完。
- **先字幕、後上架包**:字幕沒乾淨就做標題簡介=白做。
- **猜測要標記**:諧音糊掉的字給「最佳猜測」並請他確認,不裝懂硬塞。
- **標題給選項不給單一答案**,並說明為什麼推薦某個(鉤子在哪)。
- **私人 repo / 學生打不開的連結不要放進簡介**;OG 縮圖/封面風格走 BEXS 課綱風。
- **每次抓到新錯字 → 回頭加進 `references/term-correction-tables.md`**(解一次固化一次,字典越養越強)。
