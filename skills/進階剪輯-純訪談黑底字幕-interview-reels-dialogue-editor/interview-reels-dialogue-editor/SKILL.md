# Interview Reels Dialogue Editor

把「純訪談、沒有露臉、沒有可用主畫面」的素材，剪成有質感的 9:16 黑底逐字稿短片。

核心不是亂塞 B-roll，而是把訪談本身變成主畫面：說話者、句子、節奏、情緒轉折、章節、CTA 都要清楚。

## 中文 SKILL 題型關鍵詞

看到這些需求時，用這個 skill：

- **純訪談 Reels**：訪談剪 Reels、Podcast 片、沒有露臉、沒有畫面、只有聲音。
- **黑底高級字幕**：黑底字幕、逐字稿視覺、訪談字幕片、Podcast 感短片。
- **主持人/受訪者分離**：Rick Bowen 跟受訪者、HOST/GUEST、說話者不要混、誰講話要分清楚。
- **字幕完整校正**：字幕不能漏、我講話也要有字幕、不要自己省略、讀完字幕再剪。
- **9:16 IG 安全區**：Reels 安全區、手機直式、上方名字下移、底部不要被 UI 擋。
- **統一高級設計**：PingFang TC、黑金青綠、CTA、章節字、訪談品牌感。
- **Palmier 可檢查軌道**：我要看剪輯過的軌道、V1/A1 要分段、不能一整條看不出剪過。

不要用在：

- **有主畫面的長片自動切重點**：改用 `video-shorts`。
- **既有 Palmier 短影音加 HUD/封面精修**：改用 `palmier-pro-shorts-finisher`。
- **只要 YouTube 標題、簡介、封面提示詞**：改用 `rick-yt-publish`。

## 這個跟其他 skill 的差別

- `video-shorts`: 從長片自動抓重點，適合有主畫面、教學、盤面、講者影片。輸出通常是多支短影音。
- `palmier-pro-shorts-finisher`: 接續既有 Palmier 專案，做封面、HUD、字幕區、水印、片尾動畫等視覺精修。
- `interview-reels-dialogue-editor`: 這個 skill。從訪談音訊/訪談片段出發，先校準逐字稿與說話者，再建立黑底高級字幕片。適合 Podcast 感、純訪談、沒有露臉的內容。

如果任務同時提到 Palmier 和純訪談，先判斷是不是「從訪談重新剪成黑底字幕片」：

- 是 → 用本 skill。
- 不是，只是既有短片太醜要修封面/HUD → 用 `palmier-pro-shorts-finisher`。
- 是從長影片挑出幾支短片 → 用 `video-shorts`，必要時再接本 skill 做其中一支訪談型片。

## Rick 滿意的成品標準

- 9:16，1080x1920，內容全部在 IG Reels 安全區內。
- 黑底高級訪談風，不是普通字幕、不像簡報、不像廉價投資廣告。
- 主持人 `RICK BOWEN` 與受訪者名字清楚分開；Rick 用金色，受訪者用青綠色。
- 所有字體統一 `PingFang TC`。不要一段宋體、一段黑體、一段英文風混用。
- 每一句有人講話都要有字幕。不能因為句子短、語助詞、承接句就省略。
- 說話者不能混。Rick 的話不能跑到受訪者名下，受訪者的話不能變 Rick。
- 字幕斷句要像人看得懂的語意，不要機械換行。
- 有章節/情緒線，但不要塞太多標籤。底部章節字可放大，但不要搶主字幕。
- 最後要有 CTA。CTA 要延續影片語氣，不要突然變銷售頁。
- Palmier Pro 裡要看得到剪輯軌道：至少 V1/A1 分段清楚，不要一整條看不出剪過。
- 沒有聲音、聲音斷掉、某些 clip 靜音，是重大失敗。輸出前必查音軌。

## 標準流程

### 1. 先讀完整素材，不要先做畫面

1. 收到 Drive、資料夾、影片、音訊或 Palmier 專案後，先列出素材。
2. 確認這次是：
   - 純訪談 / Podcast 感，沒有露臉。
   - 有主持人與受訪者兩個角色。
   - 需要 9:16 Reels/Shorts。
3. 先完整讀取字幕/轉錄，再決定剪輯節奏。不要先用圖片、B-roll 或過場硬剪。

### 2. 轉錄與說話者分離是第一優先

先取得完整逐字稿，再做 speaker review。不要只靠聲波猜。

基本資料結構每段要有：

- `index`
- `start`
- `end`
- `speaker`
- `text`
- `audio_source`

說話者標準：

- Rick 固定顯示：`RICK BOWEN`
- 角色：`HOST`
- 受訪者用真名，例如：`粥粥`
- 角色：`GUEST`

如果自動 diarization 不穩，必須做人工驗證 UI，而不是硬做成片。

### 3. Speaker Review UI 必須好用

Rick 不應該一段一段痛苦點選後又被頁面跳走。Review UI 至少要做到：

- 每段顯示段號、時間、文字、目前 speaker。
- 點 Rick / 受訪者後立即 autosave。
- 點完自動播放下一段，減少重複點擊。
- 不要每次點完刷新整頁後跳到頂端；要保留 scroll position 或局部更新。
- 提供鍵盤快捷鍵，例如 `1=Rick`, `2=受訪者`, `Space=播放/暫停`, `N=下一段`。
- 要能從指定段號跳轉，例如 094、095、096。
- 若 Rick 說「我調整好了」，要重新讀取 label 檔並比對磁碟內容，不要假設已改。

驗證重點：

- 抽查每一次 speaker 連續切換的位置。
- 特別檢查短句、附和句、問句，這些最容易被分錯。
- 如果 Rick 指出「095 的字幕要加在 096」，要插入段落後重新 normalize index，不能只改畫面字。

### 4. 字幕不能漏，不能亂摘要

這種片是逐字稿視覺，不是摘要影片。

硬規則：

- 有聲音就要有字幕。
- 不能自行省略 Rick 的短句。
- 不能把兩個人的話合併成同一個 speaker。
- 不能把「嗯、對、沒有錯」這類承接句全刪光；只有明確無意義噪音才可移除。
- 語意不確定的字要標記給 Rick 確認，不要硬猜。

斷句規則：

- 每屏 1 到 3 行。
- 避免把固定詞拆開，例如「斐波那契」、「不用領死薪水」、「看懂市場」。
- 優先斷在語意單位，不是固定字數。
- 長句先改成自然行：
  - `交易一個月`
  - `她不再`
  - `亂猜方向`

### 5. 內容結構先定，再做視覺

訪談短片通常用這條線：

1. Hook：用問題或痛點開頭，例如「你現在交易還會緊張嗎？」
2. 痛點：怕止損、自學卡住、會畫但不會用。
3. 轉折：系統化後開始理解方向、斐波那契、關鍵位置。
4. 撞牆：連續止損、混沌期、懷疑自己。
5. 重建：接受止損、找原因、按系統做。
6. 結果：不再亂猜方向、接近夢想、不想領死薪水。
7. CTA：交易不是猜方向，是建立能重複的判斷。

不要把 B-roll 當主角。純訪談沒有畫面時，主角是「文字、節奏、聲音、沉默」。

### 6. 視覺系統

預設畫面：

- 背景：純黑或極深黑。
- 字體：`PingFang TC` 全片統一。
- Rick：金色，例如 `#E8BE64`。
- 受訪者：青綠色，例如 `#6CDEEE`。
- 主字幕：白色或依 speaker 淡色化，保持可讀。
- 章節/底部 progress：低調金色線條。
- 裝飾：細線、引號、垂直線、極淡網格即可。

不要：

- 不要紫藍漸層。
- 不要廉價紅綠 K 線。
- 不要卡片堆卡片。
- 不要每段亂換字體或色彩。
- 不要用 AI 圖片硬比喻，除非 Rick 明確要求那版風格。

建議版面：

- 左上：`RICK BOWEN / HOST`
- 右上：`粥粥 / GUEST`
- 中區：speaker label + quote mark + 主字幕
- 底部安全區內：章節名 + progress line
- 最後：CTA 黑底大字，保持同樣色彩系統

### 7. IG Reels 安全區

1080x1920 的保守安全區：

- 左右至少 `80-100px`
- 上方至少 `170px`
- 下方至少 `280px`
- 右側避免重要文字，留給 Reels 按鈕。
- 底部避免重要文字，留給帳號、文案、音訊資訊。

如果 Rick 給安全區圖，以圖為準。實作時要用常數寫進 renderer，不要憑感覺拖。

常用常數範例：

```text
SAFE_LEFT = 92
SAFE_RIGHT = 988
SAFE_TOP = 170
SAFE_BOTTOM = 1640
HEADER_Y = 174
DIALOGUE_X = 124
DIALOGUE_Y = 610
DIALOGUE_W = 832
FOOTER_TEXT_Y = 1624
FOOTER_LINE_Y = 1712
```

### 8. 音訊與時間線

輸出前必查：

- 每個 segment 都有 audio。
- A1 不可中途空掉。
- clip start/end 沒有錯位。
- 如果有靜音段，只能是明確設計的 CTA 或轉場，不可出現在訪談本體。
- 用 `ffprobe` 或 renderer log 檢查整片 duration 與 segments duration 一致。

Palmier Pro 時間線標準：

- V1/A1 要是可檢查的多段 clips。
- 每段檔名包含 index，例如 `seg_094`，方便 Rick 指錯。
- 產生新版本時另開 `.palmier` 副本，不要讓快取害 Rick 看舊版。
- open 專案後要確認 Palmier 視窗標題是最新版本。

### 9. CTA 與封面

CTA 要跟訪談主軸同一套，不要突然變硬銷售。

可用方向：

- `交易不是猜方向，是建立能重複的判斷。`
- `如果你也在交易撞牆期，先建立能重複的系統。`
- `追蹤 Rick Bowen，看更多交易成長紀錄。`

封面若 Rick 要丟 ChatGPT 網頁版設計：

- 給設計提示詞，不要自己本地出圖。
- 明確寫 1080x1920、IG 安全區、黑底、PingFang TC、Rick 金色、受訪者青綠色。
- 中文大標要短、有衝突，例如：
  - `交易一個月 / 她不再 / 亂猜方向`
  - `連續止損後 / 她反而 / 看懂市場`
  - `自學交易 / 卡住的人 / 都該看`

如果 Rick 要 Codex 直接做封面，中文字要本地後製，不要交給圖像模型精準生字。

## 常用驗收清單

做完先自己過一輪，不要等 Rick 指出基本錯。

- [ ] 逐字稿完整，沒有明顯漏句。
- [ ] Rick / 受訪者 speaker labels 已人工驗證。
- [ ] 被 Rick 指過的段號已重新讀檔確認，不是記憶中的舊值。
- [ ] 全片字體都是 PingFang TC。
- [ ] 角色名稱夠大，且位置不被上方 UI 擋。
- [ ] 所有重要文字在 IG 安全區內。
- [ ] 底部章節字夠清楚，但沒有壓到 Reels 底部 UI。
- [ ] A1 全程有聲音，除了 CTA/刻意靜音。
- [ ] Palmier 專案是新版本名稱，Rick 不會打開舊版。
- [ ] 最終回覆附上正確 `.palmier` 路徑、輸出影片路徑、review/label 檔路徑。

## 踩過的雷與修正

- **把純訪談剪成素材拼貼**：錯。沒有露臉時先用黑底逐字稿建立質感；B-roll 只當少量點綴。
- **沒有先讀完整字幕就想畫面**：錯。先讀完整逐字稿與說話者，再做節奏和版面。
- **speaker 混在一起**：重大錯誤。必須做 speaker review UI，並對 Rick 指出的段號重新讀檔。
- **Rick 的聲音沒字幕**：重大錯誤。逐字稿片不能省略主持人的短句。
- **只靠聲波分人**：不夠。聲波可輔助，但最後要用文字+聽音+人工標籤。
- **review UI 點完跳到頂端**：錯。要 autosave、保留位置、自動播放下一段。
- **Palmier 沒刷新**：另存新 `.palmier` 名稱、換新 segment folder、新 media ID，再 open 最新專案。
- **有些地方沒聲音**：輸出前檢查 A1、segment audio、duration。不要只看畫面。
- **設計沒有一致性**：統一黑底、PingFang TC、Rick 金、受訪者青綠、同一套 header/footer。
- **沒有對齊 IG 安全區**：用安全區常數，不能憑肉眼。
- **讓 GPT 生中文字封面**：容易缺筆畫。若要精準中文，改成本地/Canva/設計工具後製；若 Rick 指定丟網頁 GPT，就給它安全區與排版規格。

## 協作標準

- Rick 說「不滿意」時，先定位是哪一類錯：speaker、字幕完整度、節奏、設計、安全區、Palmier 刷新、音訊。
- 不要反覆讓 Rick 做痛苦驗證；工具要自動播放、autosave、可跳段。
- 每次改版都清楚說明版本名，例如 `Rick_Zhou_Reels_106_PingFang_TC.palmier`。
- 如果 Rick 說「先暫停」，立刻停，不要繼續剪。
- 如果 Rick 說「可以繼續」，接續最新版本，不要回到舊方向。
- 如果 Rick 說「你是不是全都下載了」，回報素材狀態，不要含糊。
- 如果 Rick 問「兩個 skill 會不會混」，要明確說明：本 skill 管純訪談黑底字幕，`palmier-pro-shorts-finisher` 管既有短影音視覺精修，`video-shorts` 管自動切重點。
